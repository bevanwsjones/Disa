\doxysubsection{Detailed Description}
An graph G(\+V, E), where V is a set of vertex indices and E is a set of undirected edges connecting the vertices in V. 

An adjacency graph G stores a pair of integer sets, V and E. V represents a list of vertex/point/node labels and E contains a set (list) of pairs, each containing two unsigned integers which are elements of V. If V is an ordered continuous set of unsigned integers it can be represented implicitly. This class thus represents G by storing the connectivity, of each vertex, to its connected neighbours rather than V and E directly. This allowing for single faster search times, similar to other \textquotesingle{}sparse\textquotesingle{} containers.

From a design point of view this class is similar to that of a sparse vector, and thus follows most of the std\+::vector paradigms, except that each \textquotesingle{}element\textquotesingle{} is dynamically sized and an ordered set of unsigned adjacency values.

@notes\+:
\begin{DoxyEnumerate}
\item Edges are undirected, and thus the edge connecting vertex i to j is the same as the edge connecting j to i.
\item Currently the graph does not support self connected edges i.\+e. edges i to i.
\item Once populated, no information about the edge indexing is stored.
\item The graph cannot contain information for connectivity between two types of vertices (i.\+e. a red set and a blue set where each set begins indexing at 0).
\end{DoxyEnumerate}

Future\+:
\begin{DoxyEnumerate}
\item Add iterators, begin(), end() etc.
\item Create arithmetic operators +,-\/, etc for unions and intersections.
\item Possible base graph class from which this class can inherit. 
\end{DoxyEnumerate}
